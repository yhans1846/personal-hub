# Personal Hub 对话框编辑页 UI 优化方案

> 适用于：
>
> - 新建/编辑日记
> - 新建/编辑 Todo
> - 新建/编辑学习计划
> - 新建/编辑阅读记录
> - 新建/编辑标签
> - 新建/编辑分类
>
> 所有编辑 Dialog 必须采用统一设计语言。

---

# 一、设计目标

当前所有编辑 Dialog 存在以下问题：

- 像后台管理系统
- 表单感过重
- Label 太多
- 字段层级混乱
- 用户是在"填表"而不是"创作"
- 与整个 Personal Hub UI 风格不一致

Personal Hub 是一个长期使用的个人知识管理系统，不是企业 OA。

编辑页应让用户感觉：

> 我要开始写东西，而不是录入数据。

整体风格参考：

- Notion
- Craft
- Bear
- Day One
- Apple Notes

关键词：

- 简洁（Minimal）
- 温暖（Warm）
- 专注（Focus）
- 沉浸（Immersive）

---

# 二、统一 Dialog 规范

所有编辑 Dialog 统一：

```
宽度：

700~860px

高度：

不超过浏览器高度的85%

圆角：

16px

Padding：

32px

背景：

var(--bg-card)

阴影：

轻阴影

Header：

固定

Footer：

固定

Body：

滚动
```

禁止：

超长 Dialog 导致整个页面滚动。

---

# 三、布局规范

采用三段布局：

```
┌──────────────────────────┐
│ Header                   │
├──────────────────────────┤
│                          │
│ Body                     │
│                          │
│ (可滚动)                  │
│                          │
├──────────────────────────┤
│ Footer                   │
└──────────────────────────┘
```

Header 和 Footer 固定。

Body 单独滚动。

---

# 四、Header 设计

不要：

```
写日记
```

改为：

```
今天

记录今天发生的事情
```

或者

```
新建任务

今天准备完成什么？
```

或者

```
阅读记录

记录一本值得回顾的书
```

Header 包含：

左侧：

标题

副标题（描述）

右侧：

关闭按钮

不要出现：

大量按钮

---

# 五、正文优先原则

正文必须成为视觉中心。

所有元数据：

日期

天气

分类

地点

标签

全部弱化。

视觉比例：

```
正文

70%

元数据

30%
```

不要让用户第一眼看到：

十几个 Input。

---

# 六、Label 规范

减少 Label。

例如：

不要：

```
标题

________________
```

改成：

```
____________________

今天发生了什么？
```

Placeholder 即说明。

除非：

法律

财务

配置页面

否则：

Label 尽量减少。

---

# 七、字段布局

不要：

```
日期

标题

心情

天气

地点

正文
```

改成：

```
今天

2026-07-12

————————————

标题（可空）

————————————

😊 😄 😐 😔 😭

☀️ 🌧️ ☁️ ❄️

📍 添加地点

————————————

Markdown 编辑器
```

形成阅读流。

---

# 八、元数据区域

元数据不要全部占满页面。

建议采用：

Chip 风格。

例如：

```
📅 今天

😊 开心

☀️ 晴

📍 深圳
```

点击：

再编辑。

默认：

仅展示。

不是全部 Input。

---

# 九、心情选择

不要：

Radio。

改成：

卡片。

例如：

```
😊

很好
```

```
😄

不错
```

```
😐

一般
```

```
😔

难过
```

```
😭

糟糕
```

选中：

绿色边框

浅绿色背景

不要出现：

Radio 小圆点。

---

# 十、天气选择

不要：

Select。

改成：

Icon。

例如：

```
☀️

🌤️

☁️

🌧️

⛈️

❄️
```

点击即可。

---

# 十一、上传图片

不要使用浏览器默认：

```
选择文件
```

改为：

DropZone。

```
┌────────────────────┐

🖼️

拖拽图片到这里

或者点击上传

支持 JPG PNG WEBP

└────────────────────┘
```

上传完成：

展示缩略图。

不要显示：

文件名。

---

# 十二、正文编辑器

正文必须最大。

建议：

最小高度：

300px

最大：

自动撑满。

不要：

只有100px。

正文是整个 Dialog 的重点。

---

# 十三、Footer

Footer 固定。

```
────────────────────

取消

保存
```

保存按钮：

Primary。

取消：

Text Button。

按钮统一：

右对齐。

---

# 十四、统一间距

一级标题：

margin-bottom

24px

模块：

24~32px

Input：

16px

按钮：

24px

禁止：

字段全部贴在一起。

---

# 十五、统一宽度

Input：

100%

不要：

一个长

一个短

Select：

统一宽度。

---

# 十六、动画

Dialog：

Fade + Scale

200ms

关闭：

Fade

不要：

突兀出现。

---

# 十七、颜色

Primary：

Accent Color

不要：

每个字段不同颜色。

心情：

可以彩色。

其他：

全部使用中性色。

---

# 十八、统一编辑体验

整个项目：

所有编辑 Dialog 必须统一。

例如：

Todo

Reading

Diary

Bookmark

StudyPlan

全部遵循：

Header

↓

元数据

↓

正文

↓

Footer

而不是：

每个模块都有自己的布局。

---

# 十九、禁止事项

禁止：

- Admin Template 风格
- Element 默认 Form 样式
- 大量 Label
- Input 堆叠
- Radio 风格
- 默认 Upload
- Footer 跟随滚动
- 内容高度不足
- 字段全部同一级
- 表格化录入体验

---

# 二十、最终体验目标

用户打开 Dialog 后：

第一眼：

看到的是内容。

不是表单。

感觉像：

正在打开一个创作工具。

而不是：

后台管理系统。

整个编辑体验应符合：

Notion

+

Craft

+

Apple Notes

+

Day One

的设计理念：

**内容优先、元数据弱化、视觉轻量、交互统一、长期使用舒适。**
