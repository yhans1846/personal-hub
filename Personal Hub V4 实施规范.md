> Version：V4.0
>
> Project：Personal Hub
>
> 技术栈：Spring Boot 3 + Vue3 + TypeScript + MyBatis Plus
>
> 更新日期：2026-07

---

# 一、文档目标

本次 V4 版本**不新增业务模块**，重点优化已有功能，提升整体产品体验、代码一致性和长期可维护性。

所有优化必须遵循以下原则：

- 不推翻现有架构
- 不改变模块划分
- 不增加无意义功能
- 保持统一设计语言
- 优先优化用户体验
- 保持代码可维护

---

# 三、产品设计原则

## 1. 内容优先

所有页面都应该突出内容，而不是按钮。

减少视觉干扰。

增加留白。

## 2. 统一体验

所有 CRUD 页面保持一致。

所有 Dialog 保持一致。

所有 Toolbar 保持一致。

所有 Header 保持一致。

## 3. 长期维护

新增代码必须：

- 可阅读
- 可维护
- 可扩展

禁止重复造轮子。

# 四、Dashboard 优化

## 当前问题

首页统计信息过多。

用户打开首页后不知道应该先看哪里。

缺少今日工作重点。

---

## 优化目标

首页应该成为：

"今天应该做什么"

而不是：

"系统有多少数据"

---

## 页面布局

```

Dashboard

┌──────────────────────────────┐
│ 今日任务                     │
├──────────────────────────────┤
│ 最近编辑                     │
├──────────────────────────────┤
│ 最近阅读                     │
├──────────────────────────────┤
│ 最近学习                     │
├──────────────────────────────┤
│ 快捷操作                     │
├──────────────────────────────┤
│ 统计卡（放最后）              │
└──────────────────────────────┘

```

# 十六、开发原则

Claude Code 在开发过程中必须遵循以下要求：

1. 不新增业务模块。
2. 不修改现有领域划分。
3. 不破坏已有架构。
4. 优先复用已有组件。
5. 保持 UI 风格一致。
6. 所有页面遵循统一布局。
7. 不引入不必要依赖。
8. 所有代码符合项目编码规范。
9. 优先优化体验，而不是增加功能。
10. 每完成一个功能，确保不影响现有模块。

---

# 最终目标

本次 V4 的目标不是增加功能数量，而是让整个 Personal Hub 在体验、交互和一致性上达到更高水平。

**一句话总结：**

> 在保持现有架构稳定的前提下，持续打磨产品体验，让 Personal Hub 更统一、更精致、更适合长期使用。
