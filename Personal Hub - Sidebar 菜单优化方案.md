# Personal Hub - Sidebar 菜单优化方案

> 更新时间：2026-07

本文档用于统一 Personal Hub 左侧导航（Sidebar）的设计规范，目标是打造符合「Notion + Linear + Raycast + Apple」设计理念的导航体验。

---

# 一、设计目标

Sidebar 不仅是功能入口，更是整个系统的信息架构。

目标：

- 层级清晰
- 扫描效率高
- 长时间使用不疲劳
- 保持简洁克制
- 符合个人知识管理产品定位

避免：

- 企业后台风格
- 功能堆叠
- 菜单过长
- 信息密度过高

---

# 二、当前存在的问题

## 1、菜单层级感不足

目前所有菜单几乎属于同一级别：

```
首页

待办

笔记

日记

阅读

学习记录

学习计划

收藏夹

文件
```

用户难以快速理解：

哪些属于知识管理？

哪些属于资源管理？

哪些属于系统管理？

---

## 2、缺少领域划分

目前虽然使用：

```
工作区

管理

统计
```

但分类仍然偏功能化。

建议按照业务领域组织，而不是数据库或页面组织。

---

## 3、菜单视觉节奏单一

当前：

- 每个菜单间距一致
- Section 标题与菜单距离较近
- 没有形成阅读节奏

建议：

Section 与 Section 保持更大的留白。

例如：

```
Section

32px

菜单

8px

菜单

8px

菜单
```

---

## 4、Active 状态过重

目前：

整块绿色背景。

容易产生传统 Admin Template 的感觉。

建议：

使用更加克制的高亮方式。

例如：

- 浅绿色背景
- 左侧 Accent Bar
- Hover 更轻
- Active 更柔和

避免过度强调。

---

## 5、Hover 反馈不足

建议：

Hover：

- 背景轻微变化
- 图标颜色同步变化
- 文字颜色同步变化

不要：

- 放大
- 缩放
- 跳动动画

保持 Apple 风格的克制。

---

# 三、推荐菜单结构

建议按照领域组织。

```
工作区

🏠 首页

────────────────

知识

📄 笔记

📓 日记

📖 阅读记录

🧠 学习记录

────────────────

规划

✅ 待办任务

🎯 学习计划

────────────────

资源

🔖 收藏夹

📁 文件管理

────────────────

管理

🏷 标签管理

📂 分类管理

⚙ 系统设置

🗑 回收站

────────────────

统计

📊 数据统计
```

优点：

- 一眼理解系统结构
- 模块职责更加清晰
- 与后端领域保持一致

---

# 四、菜单视觉规范

## 图标

推荐：

```
18px
```

线宽：

```
2px
```

不要：

24px。

避免视觉过重。

---

## 菜单高度

推荐：

```
40px
```

不要：

48px。

避免 Sidebar 过长。

---

## 左右 Padding

推荐：

```
16px ~ 20px
```

保证点击区域。

---

## 图标与文字间距

推荐：

```
12px
```

保持统一。

---

## Border Radius

推荐：

```
12px
```

保持柔和。

---

# 五、Section 规范

Section：

例如：

```
工作区

知识

规划

资源

管理

统计
```

建议：

字体：

```
12px

Medium

Uppercase（可选）
```

颜色：

使用 Secondary Text。

与菜单保持：

```
16~20px
```

间距。

---

# 六、Active 状态

推荐：

背景：

```
Primary-50
```

文字：

```
Primary-700
```

图标：

同步 Accent。

不要：

整块深绿色。

可以增加：

左侧：

```
3px Accent Bar
```

增强识别。

---

# 七、Hover 状态

Hover：

仅：

```
Background Hover

+

Text Primary

+

Icon Primary
```

动画：

```
150ms
```

即可。

不要：

scale

rotate

translate

等夸张动画。

---

# 八、最近访问（推荐）

建议增加：

```
最近访问
```

例如：

```
最近

Spring Boot 学习

Vue3 笔记

Redis 总结
```

点击即可继续阅读。

适合个人知识管理。

---

# 九、收藏（后续）

支持：

右键：

```
加入收藏
```

Sidebar：

```
收藏

⭐ Spring Boot

⭐ Vue3

⭐ Docker
```

提高访问效率。

---

# 十、响应式

当 Sidebar 收起：

仅显示：

```
Icon
```

Hover：

Tooltip。

展开：

恢复文字。

保持一致体验。

---

# 十一、统一原则

Sidebar 应体现：

> **领域导航（Domain Navigation）**

而不是：

> **功能列表（Function List）**

菜单应该帮助用户理解：

整个系统有哪些领域。

而不是：

有哪些页面。

---

# 十二、最终目标

Sidebar 应具备：

✅ 清晰的领域划分

✅ 舒适的视觉节奏

✅ 克制的交互反馈

✅ 长时间使用不疲劳

✅ 与后端模块保持一致

参考产品：

- Notion
- Linear
- Raycast
- Arc Browser
- Apple Developer Documentation

最终效果：

让 Sidebar 成为整个 Personal Hub 的核心导航，而不是传统后台菜单。
