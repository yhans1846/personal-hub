# Personal Hub - 预览页面阅读体验优化

请根据 Personal Hub 的设计理念，对 Markdown 预览页面进行整体优化。

## 项目定位

Personal Hub 是一个长期维护的个人知识管理系统，不是企业后台。

设计风格参考：

- Notion
- Linear
- Raycast
- Apple Human Interface Guidelines

页面应更加偏向：

> **文档阅读器（Document Reader）**

而不是：

> **后台管理页面（Admin Dashboard）**

目标：

- 阅读舒适
- 专注内容
- 极简克制
- 长时间阅读不疲劳

---

# 一、整体布局优化

保持目前：

```
左侧目录

+

右侧正文
```

布局。

但是整体去掉后台风格。

正文区域不要有明显的 Card 感。

正文应该更像：

- Notion
- GitHub Docs
- Apple Documentation

只保留：

- 合理留白
- 最大阅读宽度
- 自然分隔

不要出现：

- 厚边框
- 大面积 Card
- 重阴影

---

# 二、正文阅读宽度

保证：

长时间阅读更加舒适。

---

# 三、左侧目录优化

保留目录。

但是：

降低存在感。

要求：

- 更轻的颜色
- 更小的字体
- Hover 有反馈
- 当前章节高亮
- 一级、二级、三级标题层级清晰

目录应该像：

Notion 的 TOC。

不要像：

后台菜单 Sidebar。

---

# 四、Header 优化

顶部 Header 保持极简。

建议：

```
← 返回

标题

阅读工具
```

不要复杂按钮。

不要彩色背景。

不要企业后台风格。

---

# 五、阅读工具

Header 右侧增加：

```
Aa

主题

更多
```

例如：

```
← 返回

UI设计方案

Aa    ☀️    ⋯
```

---

# 六、主题切换

增加阅读主题。

支持：

## 1、跟随系统（默认）

自动继承：

Personal Hub 当前主题。

例如：

```
系统：

Dark

↓

预览：

Dark
```

或者：

```
Light

↓

Light
```

即使新开浏览器标签页，也应从本地主题配置读取。

---

## 2、浅色主题（Light）

背景：

```
#ffffff
```

正文：

```
深灰色
```

适合日常阅读。

---

## 3、深色主题（Dark）

背景：

```
深灰

接近 GitHub Dark
```

正文：

```
浅灰
```

保证 Markdown 阅读体验。

---

## 4、护眼主题（Sepia）

新增：

Sepia 阅读模式。

类似：

- Kindle
- Apple Books

背景：

浅米黄色。

正文：

深棕色。

适合：

长时间阅读。

---

## 主题切换要求

主题切换：

只影响当前预览页面。

不要修改整个系统主题。

关闭页面后恢复默认。

---

支持：

例如：

```

18

支持：

窄

1000px

1.6

暂时使用系统字体。

后续方便扩展。

支持：

- 缩放

Markdown 表格：

支持：

横向滚动。

不要：

厚重背景。

- 浅背景
- 圆角

风格接近：

Notion。

提示信息。

Information Card。

```

设计：

- 左侧强调色
- 更现代

不要默认 Markdown 样式。

# 十五、内容交互

# 十六、页面滚动

平滑滚动。

# 十七、实现要求

不要单独实现一套新的主题系统。

直接复用：

Personal Hub 的 Theme Store。

建议增加：

```
previewTheme
```

支持：

```
follow

light

dark

sepia
```

其中：

```
follow
```

为默认值。

阅读配置：

例如：

字体大小。

阅读宽度。

行高。

建议保存到：

```
localStorage
```

保证：

下次打开继续保持。

---

# 十八、UI 风格要求

整个页面保持：

- 极简
- 大量留白
- 轻边框
- 小阴影
- 阅读优先
- 不像后台
- 更像专业文档网站

最终效果应接近：

- Notion
- GitHub Docs
- Apple Documentation
- Linear Docs

而不是：

Element Plus Admin。

---

# 十九、代码要求

代码保持：

- Vue 3 Composition API
- TypeScript
- 组件职责清晰
- 阅读设置独立组件
- 主题切换独立组件
- 不修改现有 Markdown 渲染逻辑
- 尽量保持低耦合，方便后续扩展

---

# 最终目标

将当前 Markdown 预览页面升级为一个专业的文档阅读器。

它不仅用于：

- 回收站预览
- 笔记预览

未来还可以用于：

- 日记预览
- 阅读记录
- 项目文档
- 系统帮助文档
- 在线文档中心

因此需要具有良好的扩展性、统一的设计语言和优秀的阅读体验。
