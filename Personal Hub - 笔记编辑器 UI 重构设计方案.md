# Personal Hub - 笔记编辑器 UI 重构设计方案

> **版本：v2.0**
>
> 本方案用于重构 Personal Hub 的笔记编辑页面。
>
> 目标不是做一个后台表单，而是打造一个真正适合长期写作、阅读和知识沉淀的 Markdown 编辑器。

---

# 一、设计目标

当前页面存在的问题：

- 页面像后台管理系统
- 操作按钮过多，视觉重心分散
- 编辑器组件感过强
- 标题没有存在感
- 分类、标签等元信息布局不合理
- 左右分栏预览影响写作专注度
- 整体缺乏"写作工具"的氛围

新的设计目标：

> 打开页面第一眼，应该感觉是在写一篇文章，而不是填写一个表单。

参考设计：

- Notion
- Apple Notes
- Bear
- Obsidian
- Linear

设计关键词：

- 简洁（Minimal）
- 专注（Focus）
- 舒适（Comfort）
- 长期使用（Long-term）
- 内容优先（Content First）

---

# 二、整体布局

采用上下结构。

```
┌──────────────────────────────────────────────────────────────┐
│ ← 返回                              已自动保存 · ⋯            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│ 无标题笔记                                                   │
│                                                              │
│ 📂 分类   🏷 标签   ⭐ 收藏   🕒 刚刚保存                     │
│ ──────────────────────────────────────────────────────────── │
│                                                              │
│ Markdown 编辑器                                              │
│                                                              │
│                                                              │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│ 字数 1024     阅读约 4 分钟          编辑 / 预览             │
└──────────────────────────────────────────────────────────────┘
```

整个页面：

只有一个重点：

**正文内容。**

---

# 三、Header

Header 保持极简。

左侧：

```
← 返回
```

右侧：

```
已自动保存

更多菜单
```

不建议继续保留：

```
发布
```

按钮。

原因：

Personal Hub 是个人知识管理系统。

保存应采用：

自动保存。

而不是：

发布。

---

# 四、标题区域

标题作为页面最重要元素。

要求：

字体：

```
36~42px
```

粗细：

```
600
```

Placeholder：

```
请输入标题...
```

而不是：

```
无标题笔记
```

当没有输入标题时：

页面仍然保持高级感。

---

# 五、笔记元信息

标题下面增加元信息区域。

例如：

```
📂 Java

🏷 Spring Boot、Vue3

🕒 今天 14:36

⭐ 收藏

👁 阅读约 5 分钟
```

要求：

- 字体较小
- 灰色文字
- Hover 有反馈
- 点击即可修改

不要放到 Header。

元信息属于笔记本身。

---

# 六、Markdown 编辑器

编辑器作为页面主体。

要求：

去掉后台组件感。

不要：

- 粗边框
- 大圆角
- 厚阴影

编辑器应该更像：

Notion。

正文自然开始输入。

保持大量留白。

---

# 七、工具栏

工具栏默认保持精简。

仅保留常用：

```
粗体

斜体

标题

引用

代码

链接

图片

表格

列表
```

更多功能放入：

```
⋯
```

菜单。

避免工具栏占据大量视觉空间。

---

# 八、编辑与预览

取消：

左右实时分栏。

改为：

模式切换。

支持：

```
编辑

预览
```

也支持：

```
编辑 + 预览（可选）
```

默认：

进入编辑模式。

理由：

写作时不需要一直查看预览。

减少视觉干扰。

---

# 九、图片体验

图片支持：

- 拖拽上传
- 粘贴上传
- 截图直接粘贴
- 点击放大
- 删除图片

上传完成后：

自动插入 Markdown。

无需复制链接。

---

# 十、自动保存

取消：

保存按钮。

采用：

自动保存。

页面右上角显示：

```
● 正在保存...

✓ 已保存

⚠ 保存失败
```

保存状态保持轻量。

不要弹窗。

---

# 十一、页面底部状态栏

增加状态栏。

显示：

```
字数

字符数

预计阅读时间

Markdown 行数

当前模式
```

例如：

```
字数 1280

阅读约 5 分钟

编辑模式
```

---

# 十二、目录（可选）

当文章超过：

```
H2
```

自动生成目录。

目录：

右侧浮动。

支持：

点击跳转。

不要一直占据页面宽度。

---

# 十三、阅读模式

新增：

阅读模式。

特点：

- 隐藏工具栏
- 隐藏状态栏
- 隐藏元信息
- 专注阅读

快捷键：

```
Ctrl + Shift + R
```

---

# 十四、全屏模式

支持：

全屏编辑。

隐藏：

- Header
- Sidebar
- 顶部导航

只保留：

Markdown 编辑器。

快捷键：

```
F11

或

Ctrl + Shift + F
```

---

# 十五、快捷键

统一快捷键：

```
Ctrl + S

立即保存

Ctrl + P

预览

Ctrl + Shift + F

全屏

Ctrl + Shift + R

阅读模式

Ctrl + /

Markdown 帮助
```

---

# 十六、空页面体验

当正文为空。

显示：

```
开始记录你的想法……

支持 Markdown

支持图片拖拽

支持 Ctrl + V 粘贴图片
```

提升新建笔记体验。

---

# 十七、设计细节

整体：

减少：

Border。

增加：

留白。

按钮：

尽量使用：

Ghost Button。

颜色：

保持系统强调色。

Hover：

轻微背景变化。

避免：

大面积绿色按钮。

---

# 十八、响应式

宽屏：

```
最大宽度：

1000px
```

正文保持居中。

超宽屏：

不要无限拉伸。

移动端：

自动切换：

单栏编辑。

---

# 十九、未来扩展

编辑器设计需预留：

- AI 总结
- AI 润色
- AI 翻译
- AI 问答
- AI 生成目录
- Mermaid
- PlantUML
- 数学公式
- 附件管理
- 引用其他笔记
- 双向链接（Backlink）
- 引用块（Block Reference）

保证后续扩展无需重新设计页面。

---

# 二十、最终设计目标

新的笔记编辑页面应该更像：

- Notion
- Apple Notes
- Bear
- Obsidian

而不是：

- Element Plus CRUD 页面
- 企业后台表单
- CMS 内容编辑器

用户打开页面后，应立即进入"写作"状态，而不是"填写表单"状态。

整个页面的视觉重心始终围绕：

**标题 → 元信息 → 内容**

所有设计都应服务于内容创作和长期阅读体验，而不是功能堆叠。
