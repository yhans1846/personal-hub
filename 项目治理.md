# Personal Hub V2 项目治理（Project Cleanup）

## 目标

对整个 Personal Hub 项目进行一次全面治理，在**不修改业务功能、不改变接口行为、不影响数据库结构、不影响用户使用体验**的前提下，提高项目整体质量。

重点：

- 删除历史遗留代码
- 清理无用资源
- 统一代码规范
- 优化项目结构
- 更新所有文档
- 降低后续维护成本

---

# 总原则

## 绝对禁止

- 修改业务逻辑
- 修改接口行为
- 修改数据库字段
- 修改接口返回格式
- 修改页面交互逻辑
- 修改用户操作流程
- 修改功能表现

只能进行：

- 重构
- 整理
- 清理
- 统一
- 优化可维护性

所有修改必须保证：

> 功能完全一致。

# 最终目标

完成后，Personal Hub 应达到以下标准：

✅ 无死代码

✅ 无未使用资源

✅ 无重复工具类

✅ 无重复 CSS

✅ 模块职责清晰

✅ 前后端命名统一

✅ 文档与代码保持一致

✅ 所有开发规范统一

✅ 项目结构清晰

✅ 易于长期维护和持续迭代

最终形成一个**干净、规范、可维护、可持续演进**的长期个人项目，而不是随着功能增加逐渐演变成难以维护的代码库。
