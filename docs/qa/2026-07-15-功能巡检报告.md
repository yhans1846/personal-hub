## 5. 根因分析（必须修复）

各业务列表页（待办 / 学习记录 / 学习计划 / 阅读 / 日记）采用 **List + Dialog/Drawer** 编辑，`router/index.ts` **没有**注册：

- `/todos/:id/edit`
- `/study-plans/:id/edit`
- `/study-records/:id/edit`
- `/study-records/new`
- `/readings/:id/edit`
- `/diaries/new`

但下列代码仍 `router.push` 上述路径：

| 位置 | 问题 |
|------|------|
| `personal-hub-web/src/modules/dashboard/Dashboard.vue` | 计划/学习/阅读条目跳转 `.../edit`；快捷「开始学习」跳转 `study-records/new` |
| `personal-hub-web/src/store/notificationStore.ts` | 通知跳转 `todos/:id/edit`、`study-plans/:id/edit` |

笔记模块是唯一正确的「独立编辑页」模式（`/notes/:id/edit`），其他模块应对齐为：

**方案 A（推荐，改动小）**：深链改为进入列表页并打开弹窗，例如 `/todos?edit=10` 或 `/study-plans?open=2`，在 List 页 `onMounted` 读取 query 后 `openEdit(id)`。

**方案 B**：为各模块补齐 `/:id/edit` 路由页面（工作量大，与现有 Dialog 模式重复）。
