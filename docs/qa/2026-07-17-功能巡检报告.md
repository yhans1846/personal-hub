## 3. 异常清单（按优先级）

### P0-1 · `PUT /api/layout` → 500 `DuplicateKeyException`

| 项 | 内容 |
|----|------|
| 现象 | 设置页菜单/卡片开关或保存布局时失败；API 探针复现 `HTTP 500` |
| 日志 | `personal-hub-server/ph-boot/logs/personal-hub-error.log`：`Duplicate entry '1-menu' for key 'user_layout.uk_user_layout'` |
| 栈 | `UserLayoutServiceImpl.save` → `UserLayoutMapper.insert` → `LayoutController.save` |
| 前端触点 | 设置 → 菜单管理（拖拽/显隐等会 `saveMenuConfig`）；巡检中设置页多个按钮伴随 `PUT /api/layout` 500 |
| 根因判断 | `save()` 先 `selectByUserAndType` 再分支 insert/update；已存在 `(user_id, layout_type)` 时仍走 insert（竞态双请求，或查询未命中已有行） |
| 建议 | upsert（`ON DUPLICATE KEY UPDATE`）或 catch `DuplicateKey` 后改 update；并给保存加同步/防抖 |

### P0-2 · `POST /api/user/avatar` 非 multipart → 500

| 项 | 内容 |
|----|------|
| 现象 | 以 `Content-Type: application/json` 调头像接口返回 `{"code":500,"message":"服务器内部错误"}` |
| 历史日志 | 仓库根目录 `logs/personal-hub-error.log`（重启前）曾记录：`MultipartException: Current request is not a multipart request`（2026-07-17 23:18 / 23:19） |
| 接口 | `UserController.uploadAvatar` 要求 `multipart/form-data` + `@RequestParam("file")` |
| 根因判断 | 全局异常未把 `MultipartException` 映射为 400 业务错误，用户侧看到「服务器内部错误」 |
| 建议 | `GlobalExceptionHandler` 捕获 `MultipartException` / `MissingServletRequestPartException` → 400「请上传文件」；前端确保 FormData |

### P1-1 · 收藏夹深链未打开新建弹窗

| 项 | 内容 |
|----|------|
| 现象 | 访问 `/bookmarks?create=1` 停留在列表空态，**无 Dialog** |
| 对照 | `/todos?create=1` 等同规范模块会自动打开新建弹窗（Browser + 脚本均确认） |
| 代码 | `bookmark/List.vue` **未**使用 `useDeepLinkDialog`；todos/diaries/readings/study-plans/study-records 已接入 |
| 按钮本身 | 点击「新建收藏」可正常打开「新建收藏」弹窗（Browser 复核） |
| 影响 | 通知/工作台/外链若指向 `?create=1` 或 `?edit=` 会「进页但不弹窗」 |
| 建议 | 与其它列表一致接入 `useDeepLinkDialog({ openCreate, openEdit })` |
