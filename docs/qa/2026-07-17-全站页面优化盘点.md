# 全站页面优化空间盘点

> 日期：2026-07-17  
> 范围：`personal-hub-web` 全部路由页面（对照 `LIST_PAGE_SPEC.md`、一屏铺满规格、设置标签云改造）  
> 目的：排出可优化点与建议落地顺序，便于按批次改

---

## 1. 结论摘要

一屏铺满与 Product 列表 **尚未收口**：笔记 / 日记 / 学习记录 / 待办 / Dashboard 已接 composable；学习计划 / 阅读仍是手写 fill + 固定 10；收藏 / 文件 / 回收站仍是整页滚动 + `size=20`。

工具栏存在 **三套以上实现**（`ListToolbar`、学习计划/阅读自定义 toolbar、待办 emoji Tab）。设置工作台标签云已较短，可视为正面样板。

建议按：**统一 composable → 收藏/文件 fill → 抽公共 Product 组件 → 文案/Header 收敛 → 长页（统计/高级设置）** 推进。

## 4. 优化项（按优先级）

### P0 — 结构不一致 / 半迁移

| ID | 问题 | 路径 | 建议 |
|----|------|------|------|
| P0-1 | fill 两套实现 | `studyplan/List.vue` · `reading/List.vue` | 改为 `useMainContentFill` + `useFillPageSize`（10/8/6） |
| P0-2 | 收藏 / 文件仍整页滚 | `bookmark/List.vue` · `file/List.vue` | 接入 plan-page 骨架；默认 10 + pad；固定行列网格 |
| P0-3 | 工具栏碎片化 | 上述列表 + `todo/List.vue` · `ListToolbar.vue` | 抽统一 Product 工具栏（搜索/筛选/视图/导出/新建） |
| P0-4 | 笔记无 Table 切换 | `note/List.vue` | 按规范补 Table/Card + localStorage |
| P0-5 | 学习记录形态例外未文档化 | `study/List.vue` | 二选一：改 Product，或在 LIST_PAGE_SPEC 标明「时间线例外」并固定槽位契约 |
| P0-6 | 回收站遗留 | `RecycleBin.vue` | fill + 矮屏降档；图标操作；骨架屏 |

### P1 — 体验与密度

| ID | 问题 | 路径 | 建议 |
|----|------|------|------|
| P1-1 | 副标题 / 计数重复 | 各 `PageHeader`、标签管理等 | 列表页 Header 以标题为主；计数进工具栏徽章 |
| P1-2 | 面包屑与 PageHeader 双标题 | `AppLayout` + 各页 | 列表页二选一弱化 |
| P1-3 | Dashboard Hero 占高 | `Dashboard.vue` | 矮屏收成单行；资源 chips 可进组件 |
| P1-4 | Dashboard ≤900 单列 + overflow hidden | `Dashboard.vue` | 窄屏允许整页滚或降低 widget min-height |
| P1-5 | 待办 Tab / 工具栏不统一 | `todo/List.vue` | lucide + 可横向滚 Tab；可选 Table |
| P1-6 | 日记月历滚动契约 | `diary/List.vue` | `.calendar-view` 明确 `flex:1; overflow:auto` |
| P1-7 | 设置「高级」仍长 | `AdvancedSettings.vue` | 手风琴 / 子导航 |
| P1-8 | 统计页过长 | `StatsView.vue` | PageHeader；区间筛选吸顶；按可见卡片折叠图表 |
| P1-9 | 搜索空态不统一 | `SearchView.vue` | 改用 `EmptyState` |
| P1-10 | 分页边距不一致 | `ListPagination.vue` vs `plan-foot` | fill 页 compact 边距 |
| P1-11 | 收藏分页条件偏严 | `bookmark/List.vue` | `total > 0` 即显示（规范） |
| P1-12 | 分类/标签控件风格游离 | `CategoryManage` · `tag/Manage` | 对齐 el-select / ListToolbar 习惯 |

### P2 — 清理与增强

| ID | 问题 | 建议 |
|----|------|------|
| P2-1 | 学习计划 / 阅读 ~900 行 CSS 重复 | 抽 `ProductTable` / `ProductCardGrid` |
| P2-2 | `ProfileSettings.vue` 无引用（若仍存在） | 删除或并入设置 |
| P2-3 | 登录页窄屏 | `max-width:100%` 自适应 |
| P2-4 | 导出仅计划/阅读 | 有后端能力再扩笔记/待办等 |
| P2-5 | 阅读类名非标准 | 统一为 `plan-page` |
| P2-6 | 文件上传入口弱 | 工具栏主按钮 + 拖拽区次要 |
| P2-7 | Ctrl+K 与 `/search` 重叠 | 统一跳转或合并 UI |
| P2-8 | 密度 Token 与列表硬编码 padding | 审计 compact 模式 |

## 5. 已做得较好的部分

1. `useMainContentFill` / `useFillPageSize` / `resolveFillPageSize` 及单测  
2. 笔记、日记、待办、学习记录、Dashboard 的 fill 接入  
3. 学习计划 / 阅读作为 Product Table/Card 参考实现（导出、徽章、图标操作）  
4. 设置工作台标签云（菜单 / Dashboard / 统计）紧凑且无二级 Tab  
5. DashCard 体内滚动契约  
6. `useDeepLinkDialog` + 旧编辑路由 redirect  
7. 共享 `PageHeader` / `EmptyState` / `ListPagination` / `ListToolbar`  
8. 外观系统（主题、圆角、密度、内容宽度）
