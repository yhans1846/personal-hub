# 双向链接（Wiki Link / Backlink）v1

> 日期：2026-07-21 · 方案 A（前端解析 + 后端回链扫描）  
> 范围：最小可用。图谱、未创建一键新建、块引用、改标题自动改链不在本版。

## 目标

笔记可用 `[[标题]]` 互相引用；编辑可补全；预览可跳转；预览/编辑可看「谁引用了我」。由实验开关控制。

## 背景

- 正文存 `notes/{id}/note.md`，无内容列、无链接表。
- Vditor IR 不识别 wiki 语法；需自管补全与预览转换。
- `featureFlagStore.backlink` 现为 `available: false`。

## 设计

### 语法与匹配

| 项 | 约定 |
|----|------|
| 基础 | `[[笔记标题]]` |
| 别名 | `[[笔记标题\|显示名]]`（预览显示「显示名」，匹配用标题） |
| 匹配 | 当前用户、`is_deleted=0`；标题 trim 后精确相等；重名取 `updated_at` 最新一条 |
| 未命中 | 预览仍渲染为链接样式但带「失效」态，点击提示或不跳转 |

### 存储

不改表、不改路径；`[[...]]` 作为普通 Markdown 原文写入 `content` / `note.md`。

### 编辑器补全

| 项 | 约定 |
|----|------|
| 触发 | 光标前出现未闭合的 `[[` |
| 数据 | `GET /api/notes?keyword={片段}&size=8`（沿用 title+excerpt） |
| 插入 | 选中后写入 `[[标题]]`（有别名需求可后续再加） |
| 关闭 | Esc / 失焦 / 选中后 |

实现：监听 Vditor IR 输入，浮层列表；不依赖 Vditor 内置 hint。

### 预览渲染

| 项 | 约定 |
|----|------|
| 时机 | `Vditor.preview` **之前**对 Markdown 做预处理，或预览后 DOM 扫描替换 |
| 行为 | 命中 → 内链到笔记预览路由；未命中 → 失效样式 |
| 表面 | `Preview.vue`、`NoteMarkdownPreview.vue`（编辑侧栏预览） |
| 开关 | 仅 `isEnabled('backlink')` 时转换 |

### 回链 API

`GET /api/notes/{id}/backlinks`

- 鉴权：当前用户拥有该笔记  
- 逻辑：读目标笔记标题；遍历该用户未删除笔记的 `note.md`，匹配 `[[标题]]` / `[[标题|...]]`（正则，排除自身）  
- 响应：`[{ id, title }]`（可按 title 排序）  
- 上限：为防超时可限制扫描篇数（如 500），超出记日志，本版不返回分页  

### 回链 UI

- 预览页、编辑页：区块「被引用」列表，点击进对应预览/编辑  
- 无回链时短文案「暂无其他笔记引用」  
- 开关关闭时不请求、不展示  

### 实验开关

| 项 | 约定 |
|----|------|
| `available` | `true`（可切换） |
| 默认 | `false` |
| 文案 | 去掉「即将推出」；说明含补全、预览跳转、回链列表 |

### 明确不做

关系图谱 · 未创建一键新建 · `[[#块]]` · 标题变更级联改写 · `note_link` 索引表 · 文件/日记模块 wiki 链

## 验收

1. 关开关：无补全浮层、预览 `[[x]]` 当纯文本、无回链区  
2. 开开关：`[[` 可搜标题并插入；预览可点进目标笔记  
3. 笔记 B 写 `[[A的标题]]` 后，打开 A 能在回链中看到 B  
4. 不存在的标题：失效样式，不 500  
5. 他人笔记不可通过 backlinks 接口越权读取
