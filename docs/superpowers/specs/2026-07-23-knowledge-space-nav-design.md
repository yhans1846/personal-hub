# 知识空间侧栏（Knowledge Space Nav）

> 日期：2026-07-23 · 已确认（方案 B + 完整接线 + 三处覆盖 + 内联搜索 + Notion 分段 + 折中视觉）  
> 前置：`2026-07-22-note-folder-tree-design.md`、`2026-07-22-note-home-shell-design.md`（文件夹 CRUD/拖拽/删夹/直属筛选仍有效；本规格升级左栏信息架构与视觉）

## 1. 目标

将笔记左栏从「文件管理器式文件夹树」升级为现代个人知识库导航（Notion 分段结构 + Linear 交互 + Apple 留白），并命名为 **知识空间**。

成功标准：

1. 标题为「知识空间」；含搜索、首页、最近、收藏、我的空间；**无「全部」入口**。
2. 搜索可内联过滤侧栏内可见的文件夹名与笔记标题。
3. 最近、收藏可点击打开笔记；我的空间保留树能力（非只读时 CRUD/拖拽）。
4. 列表、编辑 Overlay、独立预览三处共用同一导航壳；预览为只读（无新建/拖拽/菜单）。
5. 视觉：圆角行、轻边框、弱化数量、折叠动画、hover 操作；保持 Personal Hub 极简，避免后台管理系统观感。

## 2. 范围与非目标

### 2.1 本期做

| 项 | 约定 |
|----|------|
| 覆盖面 | `/notes` 列表、编辑 Overlay、`/notes/:id/preview` |
| 架构 | 新壳 `KnowledgeSpaceNav` + 瘦身 `NoteFolderTree`（仅「我的空间」） |
| 搜索 | 前端内联过滤；不新增搜索 API |
| 最近 | 复用 `GET /api/notes/recent`，侧栏默认最多 **5** 条 |
| 收藏 | 复用笔记 `isFavorite` 查询；侧栏展示摘要（建议最多 10）；**空则隐藏整段** |
| 导航 | 保留「首页」；**去掉「全部」**；未分类仍在「我的空间」内 |
| 视觉 | 折中密度：Notion 分区文案 + 中等行高 + Linear 式 hover/选中 |

### 2.2 本期不做

- 文件夹级收藏、自定义文件夹图标/颜色
- 侧栏宽度拖拽、多知识库
- 新后端表或新搜索服务
- 「全部」入口回归（深链/旧 selection=`all` 仅兼容，不提供 UI）
- 预览页开放文件夹 CRUD/拖拽

## 3. 信息架构

自上而下：

```
知识空间                    [收起]
┌ 搜索笔记与文件夹… ───────┐
└──────────────────────────┘
🏠 首页

最近
  📄 笔记…
  （最多 5）

收藏                         ← 无数据则整段不渲染
  ⭐ 笔记…

我的空间                    [＋]  ← ＋仅非 readonly
  📁 用户文件夹…
  📁 未分类
```

### 3.1 交互

| 区域 | 行为 |
|------|------|
| 搜索 | 过滤最近/收藏/我的空间内标题；无匹配显示空态；清空恢复 |
| 首页 | `selection = home`；列表右侧回首页；Overlay/预览仅更新选中，不强制关笔记 |
| 最近 / 收藏行 | 触发 `open-note`：列表→Overlay；预览→`/notes/:id/preview`；Overlay 内切换走既有脏确认 |
| 我的空间 · 夹 | 选中过滤列表（直属）；展开/收起带动画 |
| 我的空间 · 笔记 | 同 `open-note` |
| 未分类 | 固定在「我的空间」用户夹**之后**（有笔记才显示） |
| 数量 | 默认隐藏或极低对比度；行 hover 时可显 |
| hover 菜单 | 非 readonly：⋯ 新建子夹/重命名/删除；readonly 无 |
| 收起 | 与现 Shell 一致（桌面窄轨；移动端抽屉） |

### 3.2 预览只读

- 可用：搜索、首页选中、最近/收藏/树内笔记跳转、展开收起、收起侧栏  
- 禁用：＋、拖拽、根投放区、行菜单 CRUD  

## 4. 组件与落点

| 文件 | 职责 |
|------|------|
| `KnowledgeSpaceNav.vue`（新） | 壳：标题、搜索、首页、最近、收藏、收起；向下组合树 |
| `NoteFolderTree.vue` | 仅渲染「我的空间」树（去掉首页/全部 UI 与旧总标题「文件夹」） |
| `NoteFolderTreeNode.vue` | 行视觉升级：圆角、hover 菜单、折叠 transition |
| `NoteFolderShell.vue` | 抽屉/窄轨不变；内部挂 `KnowledgeSpaceNav` |
| `List.vue` / `NoteWorkspaceOverlay.vue` / `Preview.vue` | 继续用 Shell；Preview `readonly` |

可选：最近/收藏抽成小组件，非必须。

## 5. 数据流

| 数据 | 来源 |
|------|------|
| 文件夹树 | 现有 `GET /api/note-folders/tree` |
| 最近 | 现有 `GET /api/notes/recent?page=1&size=5` |
| 收藏 | 现有笔记列表 `isFavorite=true`（分页取一页足够侧栏展示） |
| 搜索 | 纯前端对已加载摘要与树节点 title 过滤 |
| 打开/选中 | 现有 `open-note`、`v-model` Selection（`home` \| `none` \| `number`） |

不改后端契约除非实现中发现收藏列表参数缺口（若有，仅扩展 Query，不建新表）。

## 6. 视觉规范

- 行圆角 `6–8px`；搜索框同圆角 + 轻边框  
- 行间无硬分割线；分区标题 11px、次要色、段前留白  
- 选中：淡强调色底（可用 accent mix）；hover：极淡底 + 右侧操作  
- 折叠：高度/透明度约 **180ms** ease  
- 列表页背景跟应用壳；预览页跟 `--preview-*`（与现预览主题统一策略一致）  
- **禁止**：密表格、重阴影、密工具条、传统后台树控件观感  

## 7. 与旧规格关系

- `note-folder-tree-design`：CRUD、拖拽、删夹归未分类、直属筛选、备份 **仍然有效**  
- `note-home-shell-design`：默认首页、树挂笔记 **仍然有效**；左栏文案与分段以本规格为准  
- 本规格取代「左栏标题叫文件夹 / 仅树无最近收藏」的 UI 描述  

## 8. 验收清单

- [ ] 标题「知识空间」；无「全部」
- [ ] 搜索过滤夹/笔记；清空恢复
- [ ] 最近 ≤5 可打开；收藏有则显示、无则隐藏
- [ ] 我的空间：图标、折叠动画、hover 菜单（非只读）、选中高亮、数量弱化
- [ ] 列表 / Overlay / 预览共用；预览无 CRUD/拖拽
- [ ] 列表/Overlay 下拖拽与删夹等不回归
- [ ] 预览主题下侧栏仍跟 `--preview-*`

## 9. 实现提示

实现前用 `writing-plans` 出任务拆分。建议顺序：壳替换标题与分段 → 最近/收藏接线 → 内联搜索 → 树视觉与动画 → 三处挂载与 readonly → 文档（PAGE_SPEC / CHANGELOG）。
