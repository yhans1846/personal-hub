# 笔记文件夹树（列表侧栏）

> 日期：2026-07-22 · 已确认（方案 C + 列表树 + 一期含拖拽）  
> 范围：独立文件夹表、列表/编辑 Overlay/预览共用左树、CRUD、拖拽归属/层级。标签/分类仍为元数据。  
> 修订：2026-07-23 — 编辑 Overlay 与独立预览页同步展示文件夹树。

## 1. 目标

- 用户用**文件夹树**组织笔记库（单父级、一篇笔记至多一个文件夹）。
- 列表页左侧展开/收起树，点选后右侧只显示该节点下笔记；支持拖拽调整。
- 删除文件夹时笔记归入「未分类」，不进回收站。

成功标准：

1. 左树含「全部」「未分类」与用户文件夹；点选过滤列表正确。
2. 可新建/重命名/删除文件夹；新建笔记默认落入当前选中夹（「全部」→ 未分类）。
3. 可拖笔记进夹、拖夹调整父子与排序；成环请求被拒绝。
4. 删夹后该夹及子树下笔记 `folder_id = NULL`，夹本身删除。
5. 编辑 Overlay / 独立预览与列表共用文件夹树；标签/分类行为不变。

## 2. 定位与边界

| 项 | 约定 |
|----|------|
| 文件夹 | 库导航；单父级树；`folder_id` 可空 = 未分类 |
| 标签 / 分类 | 继续多选元数据；**不进树**、不承担文件夹语义 |
| 树放置 | 列表页、编辑 Overlay、独立预览页（`NoteFolderShell`）；点夹仅更新选中，不离开当前笔记；点笔记切换（编辑脏则先确认）；**预览页 `readonly`**：无首页/全部入口、无新建/拖拽/菜单（保留未分类与用户夹） |
| 文档大纲 | 编辑右栏 `EditorOutline`、预览 TOC 保持文内结构，与库导航分离 |

明确不做（本期）：多文件夹归属、文件夹权限/共享、跨用户、自动按标签生成树。

## 3. 数据模型

### 3.1 `note_folder`

| 列 | 说明 |
|----|------|
| `id` | PK |
| `user_id` | 所属用户 |
| `parent_id` | 父文件夹，根为 `NULL` |
| `name` | 显示名，同父下用户内唯一（建议） |
| `sort_order` | 同级排序，升序 |
| `created_at` / `updated_at` | 时间 |

- 不做软删：删除即物理删行（笔记已先摘掉 `folder_id`）。
- 索引：`(user_id, parent_id, sort_order)`；可选 `(user_id, parent_id, name)` 唯一。

### 3.2 `note_note.folder_id`

- 可空 `bigint`，FK 逻辑归属当前用户文件夹（应用层校验；DB 可仅索引不强制跨表 FK 若项目惯例如此）。
- 迁移：现有笔记全部 `folder_id = NULL`（未分类）。
- 列表查询：`folderId` 指定夹（**不含**子孙，除非另开「含下级」——本期**仅直属**该夹的笔记）；特殊值见 API。

深度建议：应用层限制树深 **≤ 5**（含根下第一层为 1）。

## 4. API

前缀建议：`/api/note-folders`（或 `/api/notes/folders`，实现时与现有路由风格统一，文档定一处）。

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/note-folders/tree` | 当前用户整树（嵌套或扁平+parentId，前端可组树） |
| POST | `/api/note-folders` | `{ name, parentId? }` → 创建 |
| PUT | `/api/note-folders/{id}` | `{ name }` 重命名 |
| PATCH | `/api/note-folders/{id}/move` | `{ parentId, sortOrder }` 调层级/排序；校验同用户、不成环、深度≤5 |
| DELETE | `/api/note-folders/{id}` | 见 §5 删除策略 |

笔记：

| 项 | 约定 |
|----|------|
| 列表 | `GET /api/notes` 增加 `folderId`：`省略`/`all` = 全部；`none`/`uncategorized` = `folder_id IS NULL`；数字 = 该夹**直属**笔记 |
| 创建/更新 | Body 可含 `folderId`（可空）；默认规则由前端带当前选中 |
| 移动 | `PATCH /api/notes/{id}/folder` Body `{ folderId: number \| null }`（拖拽用）；鉴权归属 |

响应树节点建议：`{ id, name, parentId, sortOrder, children?: [...] }`；可选 `noteCount`（直属，本期可选，非必须）。

## 5. 删除策略

删文件夹 `{id}`：

1. 鉴权：当前用户拥有。
2. 收集该节点及**全部子孙**文件夹 id 集合 `S`。
3. `UPDATE note_note SET folder_id = NULL WHERE user_id = ? AND folder_id IN S`。
4. 删除 `note_folder` 中 `id IN S`（子先于父或一次性 IN）。
5. **不**把笔记标 `is_deleted`，**不**进回收站。

前端确认文案：说明「文件夹将删除，其中笔记会移到未分类，笔记本身不会删除」。

## 6. 列表页 UI

```
┌─────────────┬──────────────────────────────────┐
│ 全部         │ ListToolbar（搜 / 多选 / 导入导出） │
│ ▾ 工作       │ 表 / 卡（当前 folder 过滤结果）    │
│   ▸ 项目 A   │                                  │
│   项目 B     │                                  │
│ 未分类       │  ← 固定在用户夹之后               │
│ + 新建文件夹  │                                  │
└─────────────┴──────────────────────────────────┘
```

| 交互 | 约定 |
|------|------|
| 选中 | 高亮；右侧 `fetchList` 带对应 `folderId`；分页重置到 1 |
| 展开/收起 | 仅 UI 状态（可 localStorage 记住展开集） |
| 新建文件夹 | 默认挂到当前选中用户夹下；若选中「全部/未分类」则挂到根 |
| 重命名 / 删除 | 行内菜单或右键；删除走 §5 确认 |
| 拖拽笔记 | 列表行/卡片 → 树节点（含「未分类」→ `folderId=null`）；成功后刷新 |
| 拖拽文件夹 | 树节点 → 另一节点（成为其子）或根；同级排序可随 drop 位置更新 `sortOrder` |
| 新建笔记 | Overlay 创建时带上当前 `folderId`（全部 → null） |
| ≤768 | 树默认收起，用按钮打开抽屉/面板，避免破坏一屏列表 |

列表骨架：在现有 `plan-page` 的 `plan-middle` 内改为「左树 + 右列表」，仍用 fill；左树建议固定宽约 220–260px，可后续再做拖拽改宽（非必须）。
