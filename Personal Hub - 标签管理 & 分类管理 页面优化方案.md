# Personal Hub - 标签管理 & 分类管理 页面优化方案

> Version：v2.0  
> 更新时间：2026-07-12  
> 状态：✅ 已实现  
> 目标：让标签管理、分类管理从"后台管理页面"升级为真正可管理、可统计、可扩展的资源管理中心。

---

# 一、当前存在的问题

## 分类管理

当前页面存在：

- 左侧留白过大，信息密度低
- 分类只有名称，没有任何统计信息
- 新建按钮距离内容太远
- 分类只能看到名字，没有价值感
- 排序信息比较弱
- 没有搜索
- 没有筛选
- 编辑入口隐藏
- 无法快速了解哪个分类使用最多

整体更像：

> 一个简单List

而不是

> 分类管理中心。

---

## 标签管理

相比分类稍微好一些。

但是依然存在：

- 卡片价值太低
- 使用次数没有视觉权重
- 标签颜色意义不明显
- 卡片过宽
- 空白区域巨大
- 没有搜索
- 没有排序
- 没有热门标签
- 没有统计

整个页面显得：

> 信息很少，但是占了很多空间。

---

# 二、整体设计目标

统一后台资源管理风格。

所有资源页面遵循：

```
Header
↓

统计信息

↓

工具栏

↓

资源列表

↓

分页
```

形成统一设计语言。

# 三、分类管理优化

---

## ① Header

当前：

```
分类管理

管理所有模块分类
```

优化：

```
分类管理
管理所有模块分类，可拖拽排序。

共 6 个分类
```

下面增加统计：

```
全部分类 6

正在使用 5

未使用 1
```

使用三个小统计卡即可。

---

## ② 模块切换

当前：

```
笔记分类
收藏分类
文件分类
```

建议改成：

Segment 风格：

```
——————————————

📄 笔记（6）

⭐ 收藏（4）

📁 文件（12）

——————————————
```

选中以后：

绿色底。

未选中：

白底。

更符合现代后台。

---

## ③ 工具栏

新增：

```
搜索

排序

新增分类
```

例如：

```
🔍 搜索分类...

排序：

名称

创建时间

使用次数

↑↓

                + 新建分类
```

而不是按钮孤零零跑到最右边。

---

## ④ 分类列表

不要：

```
📁 技术

排序1
```

建议做成：

```
📁 技术

18 篇笔记

创建于：

2026-06-12

————————————

✏ 编辑

🗑 删除

⋮ 更多
```

或者更简单：

```
📁 技术

18 篇

排序：1

✏

⋮
```

信息价值立即提升。

---

## ⑤ 卡片布局

现在：

```
□□□□□□

□□□□□□

□□□□□□
```

建议：

改成 Grid。

例如：

```
□ □

□ □

□ □
```

每张卡：

320px 左右。

电脑上：

2~3列。

视觉更加均衡。

---

## ⑥ 支持拖拽排序

鼠标移动：

```
☰

技术
```

左边出现：

```
☰
```

即可拖拽。

不需要单独写：

```
排序1

排序2

排序3
```

后台产品几乎都是这样。

---

## ⑦ 空状态

当没有分类：

不要空白。

建议：

```
📂

暂无分类

立即创建第一个分类

[ 新建分类 ]
```

# 十、最终效果预期

优化完成后，标签管理和分类管理页面应达到以下目标：

- 信息密度提升约 **40%**
- 空白区域减少约 **50%**
- 操作路径更加直观，常用操作可在 **2 次点击内完成**
- 支持搜索、排序、拖拽、统计等后台管理能力
- 与 Dashboard、笔记列表、文件管理形成统一设计语言
- 整体风格由传统 Admin 列表升级为现代资源管理中心（接近 Notion、Linear、Plane、GitHub Projects 的管理体验）
